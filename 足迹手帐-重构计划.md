## 一、整体架构变更

### 1.1 数据模型变更

#### ① 合集 → 路线体系重构

```prisma
// 旧：Collection（多对多地点集合）
// 新：Route（有序旅途容器）
model Route {
  id          String   @id @default(uuid())
  userId      String
  name        String                        // 路线名称
  description String?                       // 路线描述
  coverImage  String?                       // 封面图
  type        RouteType @default(collection) // collection | journey
  status      RouteStatus @default(active)  // active | completed
  
  // 旅程专属字段
  startDate   DateTime?                     // 出发日期
  endDate     DateTime?                     // 结束日期
  
  createdAt   DateTime @default(now())
  updatedAt   DateTime @updatedAt
  
  places      RoutePlace[]                  // 有序的地点列表
  user        User     @relation(fields: [userId], references: [id])
  
  @@map("routes")
}

model RoutePlace {
  id        String   @id @default(uuid())
  routeId   String
  placeId   String
  sortOrder Int      @default(0)           // 排序序号
  dayLabel  String?                        // 旅程天标注，如 "Day 1"
  notes     String?                        // 该地点在路线中的备注
  createdAt DateTime @default(now())
  
  route     Route    @relation(fields: [routeId], references: [id], onDelete: Cascade)
  place     Place    @relation(fields: [placeId], references: [id], onDelete: Cascade)
  
  @@unique([routeId, placeId])
  @@map("route_places")
}

enum RouteType {
  collection  // 合集（地点分类，无时序）
  journey     // 旅程（出游路线，有时序）
}

enum RouteStatus {
  active      // 进行中
  completed   // 已完成
}
```

#### ② 收藏即记录 — Place 升级

```prisma
model Place {
  // ... 现有字段保持不变 ...
  
  // 新增：首次记录（收藏时即可填写）
  firstImpression String?                   // 首次感受
  firstImages     String[]                  // 首次记录的照片
  firstRecordedAt DateTime?                 // 首次记录时间
  
  // 新增：情感评估
  rating          Int?                      // 评分 1-5
  wantToRevisit   Boolean  @default(false)   // 想再去
  
  @@map("places")
}
```

#### ③ 标签系统扩展


### 1.2 API 变更

| 变更类型 | 端点 | 说明 |
|----------|------|------|
| **新增** | `POST /routes` | 创建路线（合集/旅程） |
| **新增** | `GET /routes` | 获取路线列表（支持 type 筛选） |
| **新增** | `GET /routes/:id` | 路线详情（含有序地点） |
| **新增** | `PATCH /routes/:id` | 更新路线 |
| **新增** | `DELETE /routes/:id` | 删除路线 |
| **新增** | `POST /routes/:id/places` | 添加地点到路线 |
| **新增** | `DELETE /routes/:id/places/:placeId` | 移除地点 |
| **新增** | `PATCH /routes/:id/places/reorder` | 重新排序 |
| **新增** | `PATCH /routes/:id/complete` | 标记旅程完成 |
| **修改** | `POST /places` | 支持创建时写入 firstImpression + firstImages |
| **新增** | `GET /places/search?q=` | 模糊搜索地点（名称+标签） |
| **新增** | `GET /journey/timeline` | 旅程页时序数据（打卡时间线） |
| **新增** | `GET /journey/summary?year=` | 年度足迹总结 |
| **新增** | `GET /share/route/:id` | 公开查看路线 |
| **弃用** | `/api/collections/*` | 迁移到 `/api/routes` |

## 三、技术风险与注意事项

| 风险 | 应对措施 |
|------|----------|
| Collection → Route 数据迁移 | 编写幂等迁移脚本，分批执行，保留回滚路径 |
| 小程序 map 组件层级限制 | 地图区域独立，避免浮层覆盖地图，采用分区布局 |
| 重构期间破坏现有功能 | 分 Phase 实施，每 Phase 独立测试验证 |
| 包体积限制（2MB） | 重设计纯用 CSS，不加图片依赖，用 SVG inline |
| 旧 Collection API 兼容 | 保留旧端点，内部转发，过渡期后弃用 |
| 用户数据不丢失 | 收藏即记录合并不强制写感受，旧打卡数据保留 |
| 小程序性能 | 卡片列表使用虚拟列表（数据多时），避免重渲染 |
