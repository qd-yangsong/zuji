# 足迹手帐 下一阶段方向共识 v2.0

> 基于 2026-07-20 多角色头脑风暴制定
>
> 核心原则：**手帐思维、分享故事、实现高于期待**

---

## 一、核心原则

| 原则 | 含义 |
|------|------|
| **手帐思维** | 每个操作都像在写日记，不是在填表。用「手帐本」的隐喻统一产品设计语言 |
| **分享的是故事** | 不是分享产品功能，是分享「我的生活」、「我的旅程」、「我的手帐」 |
| **实现高于期待** | 技术实现要超出用户预期，而不是勉强满足需求 |
| **功能教会用户** | 不让文字说教用户，让功能本身的体验教会用户怎么用 |

## 二、7 大问题 → 新方向

| # | 问题 | 旧状态 | 新方向 |
|---|------|--------|--------|
| ① | **UI 无吸引力** | 工具思维，填表式交互，各页面割裂 | **手帐思维**，全局设计语言重构。建立统一隐喻：「手帐本」 |
| ② | **文字引导低效** | 新用户引导页说教 | **功能教会用户**：收藏后展示精美卡片预览，用户看到自己的手帐就懂了 |
| ③ | **合集方向不对** | 地点打包工具，无排序无叙事 | **路线 = 旅途容器**，产品第二条支柱。区分「合集（无序分类）」和「旅程（有序攻略）」 |
| ④ | **收藏不能记录** | 先收藏 → 再打开卡片 → 再点打卡 → 才能记录 | **合并收藏 + 首次记录**：同一页完成。基础信息在上，感受记录在下 |
| ⑤ | **「珍珠」概念抽象** | 「放入珍珠」只在概念层有意义 | **直觉化表达**：「放入我的手帐」、「记录今天的感受」 |
| ⑥ | **发现页无意义** | 静态地图看板，用户打开、看一眼、关掉 | **用地点标记的人生时间线**。时序视图（生活全景）+ 路线视图（旅行特写） |
| ⑦ | **筛选价值有限** | 按年份标签，不符合用户查找思维 | **场景化筛选词**：「全部」「常去的」「想再去」「餐厅」「咖啡店」…时间排序退到排序按钮里 |

## 三、本轮新增：更大的架构变革

### 3.1 记录升级为一级入口

**旧架构：** 底部 3 Tab — 地点 / 发现 / 我的（全是浏览）
**新架构：** 底部 4 等大 Tab — 地点 / **记录** / 旅程 / 我的

> 「记录」Tab 的图标是一个打开的笔记本 / 笔尖（不是加号）。
> 点击 → 自动定位 → 拍照写感受 → 系统匹配已有地点或创建新地点。

| 场景 | 系统行为 |
|------|----------|
| 定位匹配到已有地点（< 100 米） | 列出候选：「你是不是在这？」用户确认后直接记录 |
| 定位附近无匹配 | 创建新地点 + 首次感受，一页完成 |
| 多地点重叠（商圈） | 列出所有候选，用户选择 + 「这是新地方」选项 |

### 3.2 「我的」页面 → 使用痕迹

**旧状态：** 设置 + 反馈 + 使用指南的工具箱
**新方向：** 三层次结构

```
📔 一本手帐的封面
   ├── 头像 / 昵称 / 个性签名
   ├── 统计数据（用「厚度」感呈现）：X 个地点 / X 次打卡 / X 条路线
   ├── 贴纸拼贴区（展示已获得的稀有贴纸）
   └── 功能入口：我的路线 / 我的贴纸本 / 设置
```

> 「我的」页面不是功能面板，是**你的手帐本的封面**。使用痕迹本身就是最好的个性。

### 3.3 贴纸系统 — 情感标记 + 收集系统

**定位：** 贴纸 = 情感标记 + 收集系统 + 个人化表达

#### 三区结构

| 区域 | 名称 | 数量 | 获取方式 | 视觉 |
|------|------|------|----------|------|
| 📦 | **基础贴纸区** | 60 款 | 全量开放，渐进式展示 | 正常彩色 |
| 🏆 | **纪念贴纸区** | 5-8 款 | 旅程解锁 | 灰色诱人剪影 |
| 👑 | **荣誉贴纸区** | 10-15 款 | 成就解锁 | 金色边框 |

#### 六种情感分类

🌸 可爱 / 😜 搞怪 / 💕 浪漫 / ☀️ 美好 / 🎭 情绪化 / ☕ 场景

#### 全量开放策略

- 基础贴纸天然属于用户，不设任何障碍
- 但采用**渐进式展示**而非一次性灌输：
  - Day 1：推荐 Tab 展示 12 款精选
  - Day 3 / 打卡 5 次：自动浮现更多
  - Day 7 / 收藏 10 个：全部 60 款可浏览
- 不做黑名单，做引导。用户自己探索到其他分类本身就是惊喜

#### 贴纸应用位置

- 地点卡片角标（1-2 个，不遮挡内容）
- 打卡记录页边（装饰贴纸）
- 「我的」页面封面（贴纸拼贴区）

#### 技术实现

- 纯 SVG inline 渲染，85 款 ≈ 120KB
- 基础贴纸前端写死，纪念/成就贴纸存储为 User.sticker_unlocks JSON
- 未解锁贴纸用 CSS `grayscale(1) + opacity(0.4)` 渲染诱人剪影

### 3.4 旅程起点 — 无感植入

用户不需要知道「我在开始一段旅程」。系统在后台检测行为模式，在合适的时机轻提示：

| 触发条件 | 提示内容 |
|----------|----------|
| 2 天内打卡 3+ 个不同地点，且在同一区域 | 「这几个地方串起来，是一条不错的路线」 |
| 创建地点时选择「加入路线」 | 直接作为旅程起点 |
| 分享地点给朋友时附带「开始一段旅程」选项 | 可选开启旅程模式 |

> 所有提示都是**轻柔的卡片尾部浮出**，不是弹窗。用户可能忽略，但不会被打扰。

## 四、导航架构对比

```
旧架构                          新架构
┌──────────────┐         ┌──────────────────┐
│  地点  发现  我的 │         │ 地点  记录  旅程  我的 │
└──────────────┘         └──────────────────┘
   3 Tab 全浏览             4 Tab，记录是核心动作

旧「发现」：地图看板        新「旅程」：人生时间线
旧「我的」：工具箱          新「我的」：手帐本封面
旧：收藏 → 打卡 分两步       新：收藏即记录，一页完成
旧：筛选按年份              新：筛选按场景
旧：合集 = 打包             新：路线 = 旅途容器
```

## 七、数据模型变更概要

- **Route**：新模型，替代 Collection。包含 `type`（collection / journey）、`status`（active / completed）、`startDate`、`endDate`
- **RoutePlace**：新模型，含 `sortOrder`、`dayLabel`、`notes`
- **Place**：新增 `firstImpression`、`firstImages`、`rating`、`wantToRevisit`、`stickerIds`
- **CheckIn**：新增 `stickerIds`
- **Tag**：新增 `vibe` / `scene` 类型
- **User**：新增 `stickerUnlocks` JSON 字段
