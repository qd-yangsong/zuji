# 足迹手帐 · 完整功能梳理

## 一、产品概述

「足迹手帐」是一款微信小程序，帮助用户收藏、记录和管理去过的地方，将散落在各个App（小红书、大众点评、地图等）中的地点信息统一收纳，方便随时查找和分享。

## 五、数据流与架构

### 前端技术栈
- **框架**：Taro 4.x + React
- **语言**：TypeScript
- **样式**：SCSS
- **状态管理**：Taro Storage + pub/sub（因zustand 5.x在真机上导致无限递归）

### 后端交互
- **API请求**：封装在 services/ 目录下
- **认证**：JWT Token，401自动清除Token并提示重新登录
- **数据存储**：腾讯云COS存储图片

### 关键数据流
```
用户登录 → 获取Token → 访问地点/打卡/合集数据
创建地点 → 上传封面图 → 提交到后端 → 返回地点信息
打卡记录 → 上传图片（Promise.allSettled）→ 提交到后端 → 更新统计
分享地点 → 生成分享链接 → 非用户打开只读页面
```

## 六、设计亮点

1. **主题系统**：根据地点名称首字自动匹配6种渐变色主题，每个地点卡片都有独特的视觉风格
2. **几何图形装饰**：纯CSS绘制的几何图形（圆、方、三角），作为卡片背景装饰，无图片依赖
3. **珍珠仪式感**：打卡成功时触觉反馈，增强"放入珍珠"的仪式感
4. **卡片设计**：首字圆形徽章 + 渐变背景 + 标签展示，信息层次清晰
5. **动画效果**：登录页星光粒子动画、卡片交互动画、页面切换动画
