当前 UI 问题诊断

根本原因：COS 横版插画在卡片缩略图和详情封面中被裁剪/压缩，装饰元素跨主题混用，缺乏统一视觉规范

❶ 插画比例失真

COS 上的主题插画是 1024×768 横版场景图，被塞进卡片 160rpx 小区域和详情页封面后严重裁剪变形

❷ 装饰元素混搭

云朵、星星、月亮、爱心等装饰跨主题通用，与主题色不协调，视觉杂乱无焦点

❸ 构图层次缺失

首字徽章 + 插画 + 装饰 + 角落卡片同时出现在小区域，元素过多互相干扰

❹ 缺乏纸质温度

产品定位是"手帐"，但界面是标准 App 风格，缺少手账特有的纸质质感和贴纸趣味

新设计原则

🎨 一主题一渐变一emoji 📖 手账纸质底纹 ✨ CSS 绘制装饰 🧸 贴纸风格元素 💜 大量留白 🔄 统一圆角语言

卡片墙 · 新设计

移除 COS 横版插画，改用纯渐变 + 大号 emoji 主题图 + CSS 装饰圆。简洁、统一、可爱

实现路径 · 4阶段

从 UI 基础重构开始，逐步叠加交互体验

阶段一 · UI 基础重构

🎨 视觉系统重建

  • 主题系统重构：移除 COS 插画引用，每主题 = 渐变色 + emoji + accent色
  • 卡片墙重绘：纯渐变背景 + 大号emoji + CSS装饰圆 + 白色信息区
  • 详情页重绘：渐变封面 + 居中emoji + 移除杂乱装饰 + 贴纸风时间轴
  • 全局纸质底纹：#faf6f1 米白底色 + 微噪点纹理
  • 统一圆角语言：卡片20rpx / 按钮100rpx / 标签100rpx
  • 搜索按钮放大 + 浮动添加按钮优化
阶段二 · 卡片墙增强

🃏 交互与动效

  • 卡片入场动画：交错淡入+上移（stagger）
  • 长按卡片预览弹窗：放大显示封面+标签+最近打卡
  • 收藏爱心触觉反馈（vibrateShort）+ 弹跳动画
  • 打卡里程碑角标：3/5/10次显示铜/银/金勋章
  • 下拉刷新自定义动画：珍珠弹跳
阶段三 · 打卡仪式感

🫧 珍珠仪式

  • 打卡页珍珠可视化：大号珍珠球 + 光晕动效
  • 心情选择器：5种emoji心情 + 选中放大
  • 天气自动获取：通过 wx.getWeatherType 或手动选择
  • 提交动效：珍珠落入瓶中 + 粒子散开
  • 打卡日签：自动生成可保存的日卡（canvas绘制）
  • 连续打卡火焰：详情页显示连续打卡天数+火焰图标
阶段四 · 旅程叙事

🗺️ 足迹地图

  • 统计卡片：地点数/打卡数/城市数/总天数
  • 地图路径连线：相邻打卡点虚线连接 + 绘制动画
  • 时间滑块：拖动回放历史打卡轨迹
  • 底部信息卡：点击marker弹出地点摘要
  • 城市点亮：首次打卡城市显示点亮动效
